# AI Truth

**看清 AI 什么时候在编造内容。让 AI 更诚实——并展示它的依据。**

为 Claude、ChatGPT 和 Gemini 提供证据标签。  
显示哪些内容是**在本次对话中搜索得来**、哪些来自**记忆**、哪些是**推理**——而非一切都以同样自信的语气呈现。

## ✨ 为什么做这个项目

大语言模型经常将**事实**、**记忆回调**、**分析**和**猜测**混在一个流畅的回答中。  
问题不仅是幻觉——而是一切听起来都*同样确定*。

AI Truth 让这种差异变得可见。  
它将一个结构化的可信度提示词与一个 Chrome 扩展相结合，将证据标签直接展示在聊天界面中。

阅读[设计历程](./docs/design-journey.md)——了解框架背后的思考、错误和迭代过程。

## 📦 仓库包含什么

- **可信度提示词框架**，适用于 Claude、ChatGPT 和 Gemini
- **两个提示词版本**，适用于不同场景：
  - **精简版**：用于常驻个性化设置 / 自定义指令
  - **完整版**：用于新对话 / 研究 / 事实核查，标注更严格
- **内联证据标签**，如 `[S1]`、`[M2+R2]`、`[S3+R2+F]`、`[U+C]`
- **风险前缀**，用于高风险领域，如 `⚠法律`、`⚠金融`、`⚠医疗`
- **Chrome 扩展**，可以：
  - 在 AI 回复中可视化标签
  - 一键复制框架提示词
  - 在支持的聊天界面中注入提示词版本

## 🏷️ 框架做了什么

### 1) 输出前审查

在回答之前，模型会对关键声明执行一个**隐式 6 项检查**：

1. 事实准确性
2. 无依据的结论
3. 时效性
4. 推理漏洞
5. 缺失前提
6. 完整性

未通过检查的声明应在最终回答展示前被修正、降级或标记为不确定。

### 2) 证据标注

最终回答按证据类型标注声明，用户可以看到每条陈述实际有什么样的支撑。

| 类别 | 标签 | 含义 |
|------|------|------|
| **S**（搜索） | S1, S2, S3 | 在本次对话中实际搜索过：多源交叉验证 → 单一可靠来源 → 弱来源/间接来源 |
| **M**（记忆） | M1, M2, M3 | 稳定共识 → 可能过时 → 时效敏感，应先搜索 |
| **U**（用户） | U | 用户提供，未经外部验证 |
| **R**（推理） | R1, R2, R3 | 机械可验证 → 依赖框架 → 开放性综合推理 |
| **C**（创造） | C | 生成式构思或设计 |
| **F**（脆弱） | F | 支撑不足、矛盾或缺失 |

示例：`[S1]` `[M2+R2]` `[S3+R2+F]` `[U+C]`

高风险内容还应使用领域前缀，如：  
`⚠法律` `⚠金融` `⚠税务` `⚠医疗` `⚠安全` `⚠合规` `⚠工程`

## 🧠 应该用哪个版本？

包含两个版本：

| | **精简版** | **完整版** |
|---|---|---|
| **长度** | 约 400 tokens | 约 2500 tokens |
| **适用场景** | 个性化设置 / 自定义指令 / 常驻使用 | 新对话 / 研究 / 事实核查 / 更严格标注 |
| **行为** | 更轻量、更快，默认覆盖良好 | 规则更明确，边界控制更强 |
| **权衡** | 长对话中更容易漂移，更多边缘情况交由模型判断 | 一致性更好，但对大多数个性化设置来说太长 |

**建议：** 日常使用从**精简版**开始。当你需要更严格的边界、更强的标注和更少的漂移时，使用**完整版**。

提示词文件：

- **精简版** → [`model_instructions/prompts-v7-compact.md`](model_instructions/prompts-v7-compact.md)
- **完整版** → [`model_instructions/prompts-v7-full.md`](model_instructions/prompts-v7-full.md)

设置方式：

- **Claude** → 设置 → 自定义 Claude
- **ChatGPT** → 设置 → 个性化 → 自定义指令
- **Gemini** → 风格/偏好设置区域

## 🧩 Chrome 扩展

### 安装

1. 打开 `chrome://extensions/`
2. 启用**开发者模式**
3. 点击**加载已解压的扩展程序**
4. 选择 `extensions/chrome` 目录
5. 打开 Claude、ChatGPT 或 Gemini

支持的平台：

- **Claude**
- **ChatGPT**
- **Gemini**

### 功能

#### 🎨 标签可视化

当 AI 输出如 `[S1]`、`[M2+R3]` 或 `[S3+R2+F]` 的标签时，扩展会按可信度级别高亮段落。

提供两种显示模式：

- **简洁模式** — 清爽的徽标标签（`已验证`、`注意`、`参考`、`警告`）
- **审计模式** — 更强的颜色处理，悬停可查看标签药丸和解释

#### 📋 复制提示词

从弹窗中一键复制当前框架提示词。

#### ⚡ 提示词注入

在支持的聊天界面中注入一个**可信度提示词**选择器，无需离开页面即可选择并插入提示词版本。

## 🔍 为什么这个项目不同

这个项目不只是要求模型"更加谨慎"。  
它改变的是**用户能看到什么**。

- **不同于展示模型如何推理的方法，本项目展示的是每个结论站在什么基础上。**
- **不同于输出类概率分数的置信度方法，它使用分类标签，不假装是经过校准的。**
- **它将来源类型与推理类型分开。**
- **它增加了 UI 层，使框架在实际聊天流程中可见，而非藏在提示词里。**

## ⚠️ 已知局限

详见 [limitations.md](limitations.md)。

主要包括：

- 自检仍然继承模型自身的盲区
- 长对话会削弱提示词遵循度
- 本框架仅在前沿模型上测试过；较弱的模型可能错误标注声明，降低可靠性
- 平台 DOM 变更可能导致 UI 注入失效
- 标签提升的是可审计性，不是保证真实性
